Acerca de mí

Información sobre mí

Me llamo Caio Cesar Gomes soy programador de Aplicaciones web y de escritorio con 2 años de experiencia. También soy técnico de Sistemas Microinformáticos y Redes desde 2022. Empecé a programar a los 15 años viendo tutoriales en YouTube, al investigar un poco aprendí el funcionamiento de lenguajes como HTML y CSS, con los cuales practiqué mucho llegando a hacer un proyecto escolar. Posteriormente, me interesé en Python ya que era un lenguaje muy recomendado para novatos que apenas se introducían al mundo de la programación.

A lo largo de mi experiencia he trabajado en diversos proyectos, la mayoría hechos con Python y sus librerías como Django para páginas web o PyQt para interfaces gráficas de escritorio. Aunque mi lenguaje preferido sea Python, no descarto la posibilidad de trabajar en proyectos empleando otros lenguajes ya que siempre estoy dispuesto a aprender. Siempre intento mantener buenas prácticas al desarrollar mis proyectos y procuro organizar bien mi área de trabajo y mis objetivos antes de emprender uno nuevo.

Me considero una persona muy responsable, detallista, reflexiva y flexible. Me gusta mucho aprender y valoro mucho todos los consejos y explicaciones que me ayuden a mejorar como profesional en el sector. En un futuro aspiro a poder trabajar en alguna de las empresas más reconocidas del sector como Google, Amazon, Facebook... donde espero poder desarrollar y exprimir mis habilidades profesionales al máximo y colaborar en un grupo de trabajo.

Acerca mi trabajo

Github

Siempre uso un sistema de control de versiones para desarrollar mis proyectos para así optimizar y mejorar la calidad del código. En mi caso, uso git y github para lograr este cometido. Tengo experiencia con la plataforma de github siendo este el repositorio de todos mis proyectos tanto privados como públicos y conozco los comandos básicos de git para controlar los cambios en el código para posteriormente subirlos al repositorio.

Entornos virtuales Python

En mis proyectos escritos en Python siempre residirá un archivo "requirements.txt" en el cual se especifican los modulos y librerías necesarias para que el código funcione correctamente. Esta es una buena práctica de Python documentada en PEP-405.
